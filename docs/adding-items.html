Adding items

如果你的应用程序动态地增加新的内容，Isotope提供了几种方法来添加项目。If your application dynamically adds new content, Isotope provides several methods to add items.

See Demo: Adding items.

addItems method

addItems方法添加一个新的内容给一个Isotope容器。应用适当的样式使该项目可以被定位，且排序数据可以被检索。The addItems method adds new content to an Isotope container. This applies the proper styles to the items so they can be positioned and any sorting data is retrieved. 但是新的内容将不会被过滤、排序、或正确放置，也不会被添加到容器元素中。But that’s it. The new content will not be filtered, sorted, or positioned properly, nor will it be appended to the container element.

var $newItems = $('<div class="item" /><div class="item" /><div class="item" />');
$('#container').append( $newItems ).isotope( 'addItems', $newItems );

insert method

更可能的是，你要使用insert方法，避免addItems方法丢失的那些东西。More likely, you want to use the insert method, which does everything that addItems misses. insert方法会追加内容到容器，过滤的新内容，排序所有的内容，reLayout 触发器会使所有项目都合理布局。insert will append the content to the container, filter the new content, sort all the content, then trigger a reLayout so all item elements are properly laid out.

var $newItems = $('<div class="item" /><div class="item" /><div class="item" />');
$('#container').isotope( 'insert', $newItems );

appended method

appended方法是一种方便的方法将新的内容触发addItems，仅将新的内容放置在布局底部。如果你确定只想把新的内容添加到最后，而且不使用过滤或排序，这个方法是最好的方法，使用无限滚动。The appended method is a convenience method triggers addItems on new content, then lays out only the new content at the end of the layout. This method is useful if you know you only want to add new content to the end, and not use filtering or sorting. appended is the best method to use with Infinite Scroll.

See Demo: Infinite Scroll.

See also Infinite Scroll with filtering or sorting

Prepending

因为Isotope的排序功能，在容器前加入的功能不能像预期的那样。然而，它相当容易复制，想在容器前面加上新的内容，你可以重新收集所有的项目元素，用reloaditems方法更新排序。会触发使用原有的DOM顺序重新布局。Because of Isotope’s sorting functionality, prepending isn’t as straight forward as might expect. However, it can be replicated fairly easy. After prepending new content to the container, you can re-collect all the item elements and update their sorting order with the reloadItems method. Then trigger a re-layout, with the original DOM order.

var $newItems = $('<div class="item" /><div class="item" /><div class="item" />');
$('#container').prepend( $newItems)
  .isotope( 'reloadItems' ).isotope({ sortBy: 'original-order' });
